# 输出改造清单 Output Refactor Checklist

## 1. 改造目标

当前输出的问题不是“少”，而是“散、重、难取舍”。  
从运营商视角，真正有价值的不是很多张图，而是以下几类信息：

1. 这次事件影响多大，是否值得马上处置。
2. 攻击源是否集中，是否存在明显团伙或同源特征。
3. 来源地域、来源运营商、入口节点是否有明显热点。
4. 时间趋势是持续打、脉冲打，还是阶段性切换。
5. 能否快速形成封堵、牵引、上报、通报的依据。

因此本次改造原则是：

1. 减少重复产物。
2. 文件名中英文结合，可直接被人理解。
3. 保留“总览 + 聚类 + 来源 + 趋势”四类核心输出。
4. 把偏研究型、重复型、自动化处置型文件从默认输出中移除。

## 2. 默认保留的输出

### 2.1 核心报告

- `00_分析摘要_summary*.json`
  - 给页面、接口、自动化系统消费。
- `01_总览报告_overview_report*.md`
  - 给人阅读的主报告，作为单次分析的主入口。

### 2.2 核心数据

- `02_源IP画像_source_profiles*.csv`
  - 每个源 IP 的关键画像和分类结果。
- `03_源聚类分析_source_clusters*.csv`
  - 聚类后的团伙级结果，适合看“是否为同源打击”。
- `04_入口节点_entry_hotspots*.csv`
  - 哪些入口节点最承压。
- `05_来源地域_source_geo*.csv`
  - 来源地域热点。
- `06_来源对象_source_monitor_objects*.csv`
  - 来源监测对象热点。
- `07_时间分布_time_distribution*.csv`
  - 时间分桶数据，便于二次分析。

### 2.3 核心图表

- `10_攻击总览_attack_overview*.png`
  - 分攻击类型时保留，适合管理和汇报。
- `11_源风险看板_source_risk_dashboard*.png`
  - 看高风险源规模、强度、集中度。
- `12_运营视角看板_operator_dashboard*.png`
  - 看地域、对象、入口节点热点。
- `13_总体画像雷达_overall_profile_radar*.png`
  - 看整体风险层级画像。
- `14_疑似源雷达_suspect_source_radar*.png`
  - 看疑似高风险源差异。
- `15_时间趋势_attack_timeline*.png`
  - 看攻击走势。
- `17_分类型画像_type_profile_radar*.png`
  - 仅在分攻击类型分析时输出。

## 3. 从默认输出中移除的内容

以下内容并不是没价值，而是不应再作为默认输出：

- `source_distribution_dashboard`
  - 已经证伪，不直观，且容易误导。
- `cluster_radar_chart`
  - 与聚类 CSV、疑似源雷达、总体画像雷达存在明显重复。
- `top_attacker_radar_chart`
  - 与疑似源雷达面板重复。
- `attack_blacklist`
  - 偏自动化处置产物，不适合作为默认运营输出。
- `suspicious_sources`
  - 与源画像 CSV 重复。
- `overall_attack_situation`
  - 与总览报告重复。

## 4. 已落地改造

### 4.1 命名规范

已切换为“序号 + 中文 + 英文”的文件命名方式，例如：

- `01_总览报告_overview_report.md`
- `03_源聚类分析_source_clusters.csv`
- `12_运营视角看板_operator_dashboard.png`

好处：

1. 文件夹里按序号自然排序。
2. 运营、安全、研发都能大致看懂。
3. 页面展示时也更直观。

### 4.2 默认输出集缩减

主链路默认不再生成以下产物：

- 聚类雷达图
- Top 攻击源雷达图
- 黑名单 CSV
- 可疑源 CSV
- 冗余态势说明 Markdown

## 5. 运营视角下后续还应继续优化的点

### P0

- 在总览中增加“是否建议处置”的结论字段。
- 增加“入口承压 Top-N”与“受影响监测对象 Top-N”的摘要结论。
- 在页面中区分“汇报视图”和“分析视图”。

### P1

- 对聚类结果补充“疑似同团伙、疑似同运营商、疑似同地区”解释。
- 对趋势图补充攻击阶段分段说明。
- 增加“同一来源在多次事件中的重复命中”展示。

### P2

- 输出“建议动作”栏目：
  - 封堵
  - 清洗牵引
  - 上游协同
  - 持续观察
- 接入威胁情报库后，给出“历史命中次数、最近命中时间、情报可信度”。

## 6. 结论

这次改造的方向不是继续加图，而是把默认输出压缩为少数几个高价值产物。  
对运营商来说，最重要的是“是否需要处置、从哪里来、是否集中、是否会持续、如何协同”，而不是看到很多技术细节图后自己再去拼结论。
