# 威胁情报页面 UI 设计方案

> 版本: v1.0 | 日期: 2026-04-17 | 状态: 待评审

---

## 1. 设计理念

当前 `/intel` 页面把 Dashboard 概览和事件明细混在同一个长页面里，信息层次不清晰。
作为"威胁情报库"的呈现入口，页面体系应该像情报中心一样分层：

```
宏观态势 → 维度钻取 → 事件详情 → 源IP画像
```

用户先看到全局态势（"最近攻击压力如何、哪些维度最活跃"），
再按需下钻到具体维度（"哪些目标反复被攻击、哪些团伙活跃"），
最后才进入单条事件或单个IP的完整分析。

---

## 2. 页面体系总览

```
/intel                        威胁情报总览（态势感知 Dashboard）
  ├── /intel/events            事件列表（支持筛选、搜索、分页）
  │    └── /intel/events/:id   事件详情（已有，需优化）
  ├── /intel/sources           攻击源排行（重复攻击IP/团伙/网段）
  │    └── /intel/sources/:ip  源IP画像详情页（新增）
  └── /intel/assets            情报资产管理（黑白名单、标签、反馈）
```

底部导航栏统一入口，四页 Tab 切换：**总览 / 事件 / 攻击源 / 资产**

#### 新增 API

```
GET /api/v1/intel/events
  ?severity=critical,high          # 严重级别多选筛选
  &attack_type=UDP Flood           # 攻击类型筛选
  &target_ip=10.0.1.5              # 目标IP模糊搜索
  &target_mo=核心网关              # 监测对象搜索
  &time_range=24h|7d|30d|custom    # 时间范围
  &start_time=2026-04-01           # 自定义开始时间
  &end_time=2026-04-17             # 自定义结束时间
  &sort_by=priority|time|severity|bps  # 排序字段
  &sort_order=desc|asc             # 排序方向
  &page=1                          # 页码
  &page_size=20                    # 每页条数

响应:
{
  "total": 187,
  "page": 1,
  "page_size": 20,
  "severity_summary": {"critical": 12, "high": 34, "medium": 89, "low": 52},
  "items": [...]
}
```

#### 交互行为

- **点击行/查看详情**: 跳转到 `/intel/events/{event_id}`
- **筛选栏**: 实时筛选，不跳页
- **排序**: 点击表头切换升序/降序
- **统计摘要条**: 跟随筛选条件动态更新

### 3.3 事件详情页 `/intel/events/:id`（已有页面优化）

**当前状态**: 已有深色主题的事件详情页，包含时序趋势、高风险源明细、分层画像、地理与来源对象、入口节点。

**优化点**:

1. **顶部增加"研判摘要"卡片**（来自 `judgement` 字段）
   - 处置建议
   - 关键发现（findings 列表）
   - 证据摘要（黑名单命中数、白名单命中数、标签命中数）

2. **增加面包屑导航**
   ```
   总览 > 事件列表 > EVT-001
   ```

3. **高风险源表格增加交互**
   - 点击源 IP 跳转到 `/intel/sources/{ip}` 查看画像

4. **增加"返回列表"按钮**（当前已有，保留）

#### 优化后布局变化

```
原:  [返回Dashboard] → [标题+统计] → [趋势+源明细] → [画像+地理+入口]

新:  [面包屑] → [标题+统计] → [🆕 研判摘要卡片] → [趋势+源明细] → [画像+地理+入口]
                                    ↓
                              处置建议: "优先进入清洗联动..."
                              关键发现:
                               - 高风险源共 42 个
                               - 主要攻击类型: UDP Flood
                               - 3 个命中黑名单
                              证据: 黑名单3 / 白名单1 / 标签5
```

## 5. 视觉风格统一

### 5.1 当前问题

- `/reports` 系列使用暖色米白风格
- `/intel` 总览使用暖色风格
- `/intel/events/:id` 使用深色风格

### 5.2 统一方案

建议威胁情报体系统一使用**暖色明亮风格**（与现有 `/intel` 总览一致），
事件详情页保持深色风格不变（深色更适合展示密集的数据表格）。

| 页面 | 风格 | 背景 | 主色 |
