## 项目概述

**DDoS 攻击溯源分析器** - 基于 NetFlow 数据的智能化分析工具

### 项目目标
- **自动识别攻击源**：从海量 NetFlow 数据中自动识别 DDoS 攻击源 IP
- **特征分析**：对每个攻击源进行多维度行为特征分析
- **指纹聚类**：识别僵尸网络团伙或使用相同工具的攻击群体
- **路径溯源**：追踪攻击流量从哪个骨干接口进入被防护网络
- **可视化报告**：生成结构化的分析报告和可视化图表

### 分析粒度
以源 IP（`src_ip_addr`）为单位进行聚合分析，每个源 IP 作为一个独立的分析实体。

## 关键决策点

1. **技术栈选择**：
   - 主框架：Python + Pandas（向量化计算）
   - 数据源：ClickHouse 数据库
   - 聚类算法：HDBSCAN（优先）→ DBSCAN → MiniBatchKMeans（兜底）
   - 可视化：Matplotlib
   - 性能要求：支持百万级 NetFlow 记录处理

2. **数据处理策略**：
   - 向量化计算：避免逐行 Python 循环
   - 内存保护：超万样本自动降采样
   - 动态基线：先排除攻击 IP 再计算基线
   - 分批查询：大数据量时使用分批查询

3. **异常检测策略**：
   - 多因子评分：PPS/BPS 包大小、突发模式、行为模式
   - 加权总分：100 分制，动态阈值
   - 分层分类：4 个置信度区间

4. **数据源集成**：
   - 直接从 ClickHouse 查询，避免数据导出
   - 基于 dst_ip + dst_mo_code 组合过滤
   - 使用 parser_rcv_time 作为主要时间字段

## 风险评估

| 风险 | 等级 | 应对措施 |
|------|------|---------|
| 大数据处理性能问题 | 高 | 向量化计算 + 采样策略 + 三级算法降级 |
| ClickHouse 查询超时 | 中 | 分批查询 + 优化查询条件 |
| 误报率过高 | 中 | 动态阈值 + P95 保守策略 |
| 聚类结果不稳定 | 中 | 使用 RobustScaler 标准化 |
| 缺乏测试数据 | 低 | 生成模拟 NetFlow 数据进行测试 |
